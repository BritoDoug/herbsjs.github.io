Get Started

Domain that matters

Code your domain first using Herbs and the infrastructure will follow so. stop to spend time with redundant code. At same time we are flexible, for advanced scenarios you can provide your own code and change the behaviour.

Keep it clean

Our main concern is maintainability. Create and evolve your code knowing your project will not become a big ball of mud even on a multi year project. Concepts like Clean Architecture and Clean Code are at its core.

Auditable and Secure

Enterprise grade features are standard on Herbs even for simple applications. Authorization and auditing are be available out of the box. You can still opt-out if needed.

Suma helps with single value validations. Extensible, test covered and errors code only! Suma does not validate schema or objects, just single values. For schema validation take a look at herbjs/gotu.

const { validate } = require('suma')
const value = null
const validations = { presence: true }
const result = validate(value, validations) 
/* {
    value: null,
    errors: [{ cantBeEmpty: true }]
} */

Gotu Kola helps define your business entities (*) (*) Entities: they are the first natural place we should aim to place business logic in domain-driven applications.

const { entity, field } = require('gotu')    
const itemBuilder = 
    entity('Item', {
        id: field(Number),
        description: field(String)
    })
const item = new itemBuilder()
item.id = 123
item.description = "example"
// gotu use suma inside validate() 
item.validate()
item.errors // example: { id: [ wrongType: 'Number' ], description: { monthlyCost: [ wrongType: 'String' ] } }
item.isValid() // false

Uniform, auditable and secure use case javascript library. Influenced by Clean Architecture and Trailblazer

const {
  Ok,
  Err,
  usecase,
  step,
  ifElse } = require('buchu')
const dependency = {
    ItemRepository: require('../itemRepository'),
}
const addOrUpdateItem = (injection) =>
  usecase('Add or Update an Item on a to-do List', {
    // Input/Request type validation 
    request: { listId: Number, item: Item },
    // Authorization Audit  
    authorize: (user) => user.isAdmin ? Ok() : Err(),
    // Dependency Injection control
    setup: (ctx) => {
      ctx.di = Object.assign({}, dependency, injection)
    },
    // Step audit and description
    'Check if the Item is valid': step((ctx) => {
        //...
        return item.validate() // Ok or Error
    }),
    'Check if the List exists': step(async (ctx) => {
        //...
        return Ok()
    }),
    // Conditional step
    'Add or Update the Item': ifElse({
      'If the Item exists': step(async (ctx) => {
          //...
          return Ok(newItem)
      }),
      'Then: Add a new Item to the List': step(async (ctx) => {
          //...
          // Ok or Error
          return ctx.ret = await itemRepo.save(item)
      }),
      'Else: Update Item on the List': step(async (ctx) => {
          //...
          // Ok or Error
          return ctx.ret = await itemRepo.save(item)
      })
    })
  })
